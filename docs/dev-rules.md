# KASANE 開発ルール

## 基本方針

KASANEは、推し活と教養を接続する「1日1枚のシェアカード生成器」として開発する。

初期段階では、Home、Library、Oshi、SNS、コメント、ログイン機能などを広げすぎない。

まずは以下に集中する。

- 古典・絵画・詩の断片を美しくカード化する
- 推しっぽさを重ねる
- 日本の伝統色・余白・ジャパンディ感を活かす
- SNSでシェアしたくなる画像を生成する

## 開発上の注意

- 大改修より差分実装を優先する
- 既存UIを壊さない
- 一度に複数機能を増やしすぎない
- 文言は安直にしない
- 和風に寄せすぎない
- 「推し活」をぼかしすぎない
- アプリ内コミュニティ機能は初期実装しない
- ログイン機能は初期実装しない
- SNS機能はアプリ内に持たず、外部SNSへの共有を前提にする

## 文言方針

- 世界観はUIで見せる
- 言葉は説明する
- 変に婉曲的な表現は避ける
- 率直で、でも少し美しい表現にする
- 「OSHI」「しおり」など安直な命名は避ける

## 報告ルール

Claudeは実装後、必ず `/docs/claude-report-template.md` の形式で報告する。

ChatGPTにレビューを依頼する場合は、`/docs/chatgpt-review-template.md` の形式を使う。
